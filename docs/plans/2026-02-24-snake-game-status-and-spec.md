# 皮皮狗贪吃蛇：项目技术规格与现状说明书 (2026-02-24)

## 1. 项目概况
本程序是一个高度适配移动端与桌面端的“进化版”贪吃蛇游戏。核心特色在于**非栅格化平滑移动**、**重力感应控制**以及基于**像素级精灵切片**的美术表现。

## 2. 核心架构说明

| 模块 | 文件 | 职责说明 |
| :--- | :--- | :--- |
| `入口管理` | `js/game.js` | 负责 `GameLoop` (requestAnimationFrame)、游戏状态机 (menu/playing/gameover)、摄像机跟随、窗口缩放适配。 |
| `资源管理` | `js/assets.js` | 集中管理精灵图 (SpriteSheet) 的加载与坐标切片映射。 |
| `玩家逻辑` | `js/player.js` | 处理蛇身路径点记录 (`pathHistory`)、运动插值、体型缩放算法。 |
| `地图物理` | `js/map.js` | 维护瓦片地图、处理像素级碰撞判定（致命/非致命阻挡）、绘制地形层。 |
| `道具系统` | `js/props.js` | 负责 Bug (瓢虫) 的 AI 追踪行为、道具生成概率（受难度阶梯影响）、碰撞效果触发。 |
| `输入适配` | `js/input.js` | 包含重力感应校准逻辑、虚拟摇杆、鼠标及键盘的统一输入分发。 |

## 3. 关键技术细节

### 3.1 平滑运动逻辑
蛇身不锁定在 150x150 的瓦片网格内，而是根据 `Player.pathHistory` 记录的坐标点进行线性插值渲染。
*   `recordDistance`: 两个记录点之间的最小像素间隙（当前为 2px）。
*   `segmentSpacing`: 在渲染时，每节身体选取的历史点索引差值，随体型动态缩放。

### 3.2 碰撞检测规范
*   **致命 (Lethal)**: 边缘围栏、岩石、红瓢虫、咬住 3 节以后的身体。
*   **阻挡 (Solid)**: 树木、蘑菇房、花坛、水面 (由于重力回弹逻辑，水面不直接致死)。
*   **动态难度**: 每获得 500 分，瓢虫的追踪速度增加 30px，Buff 生成率降低约 5%。

## 4. 移动端适配参数
*   `viewScale`: 移动端默认 0.75，增加视野覆盖。
*   `gravity`: 默认校准基线，dBeta/dGamma 超过 4 度开始产生位移分量。

## 5. 待办扩展 (Superpowers 路线图)
1.  **磁铁 (Magnet)**: 自动吸取周围 300px 内的骨头。
2.  **慢动作 (SlowMo)**: 吃到特定道具后大幅减慢时间流速，方便躲避红瓢虫。
3.  **无缝场景**: 将背景纹理 (`scene.png`) 离屏渲染以提升极长蛇身下的性能。
